# 移动端触摸 QA 指南

本游戏支持移动端点击，使用 `pointerup` 和 `touch-action: manipulation`。推荐横屏体验。

## 1. 必测设备

- iPhone Safari 横屏。
- Android Chrome 横屏。
- 桌面 Chrome 设备模拟器。
- 桌面浏览器鼠标点击。

## 2. 热区检查

打开游戏左上角 `热区` 按钮，检查所有绿色区域是否覆盖对应物件。热区不应该小到难以点击。

重点检查：

1. 开场贝壳。
2. 海边旧信。
3. 海边发光贝壳。
4. 工坊老工匠。
5. 工坊工作台。
6. 夜粥街空凳子。
7. 夜粥街粥锅。
8. 夜粥街酱料。

## 3. 最小热区建议

- 移动端按钮实际像素建议不小于 44px。
- 以 16:9 画面百分比估算，热点宽高不建议低于 8%。
- 细小物件可以用更大的透明点击区域包住。

## 4. 失败处理

如果手机上点不到：

1. 打开热区调试。
2. 在 `game.js` 修改对应 hotspot 的 `x/y/w/h` 百分比。
3. 运行 `node tools/check-touch-targets.mjs`。
4. 重新打包。

## 5. 竖屏策略

竖屏不强行塞游戏画面。当前版本会显示“请横屏进入海城”。这是刻意设计，因为分镜和构图都依赖横向画幅。
